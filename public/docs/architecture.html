Thread Model

VPS는 Main Thread(렌더링+UI)와 Worker Thread(s)(시뮬레이션)를 엄격히 분리합니다.

ThreadResponsibilitiesConstraints
Main React UI, Three.js 렌더링, 사용자 입력, 카메라 제어 물리 계산 금지, SharedArrayBuffer 읽기만
Worker 물리, 충돌, Dijkstra, Lock, Transfer, 로깅 DOM/Three.js 접근 금지, SharedArrayBuffer 쓰기

SharedArrayBuffer Memory Layout

고빈도 데이터(차량 위치 등)는 postMessage 대신 SharedArrayBuffer로 zero-copy 통신합니다.

Worker Distribution

Fab들은 라운드 로빈으로 Worker에 분배됩니다. 예: 110 Fabs, 4 Workers → Worker당 ~28 Fabs.

Multi-Fab Grid System

하나의 맵을 M×N 격자로 복제하여 동시에 다양한 파라미터 조합을 테스트합니다.

Grid Creation Flow

  1. loadCFGFiles(mapName) — 원본 맵 로드 (nodes.cfg, edges.cfg)
  2. getNodeBounds(nodes) — 맵 경계 계산
  3. createFabInfos(countX, countY, bounds) — Grid 정보 생성 (10% 간격)
  4. 각 Fab에 독립 노드/엣지 복제 (ID offset = fabIndex × 1000)
  5. Worker에 sharedMapData로 원본 맵 전달 → Fab별 offset 적용

Key files: src/utils/fab/fabUtils.ts, src/store/map/fabStore.ts
