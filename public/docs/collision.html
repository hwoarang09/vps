Collision DetectionHybrid SAT + 1D Rail 최적화 — 100k+ 차량 O(1)/vehicle 실시간 충돌 감지

1. 개요

VPS Collision Detection은 100,000대 이상의 차량이 동시에 운행하는 대규모 AMHS 시뮬레이션에서 프레임당 O(1)/vehicle 충돌 감지를 달성하는 핵심 시스템입니다.

핵심 혁신: Hybrid 전략

  • LINEAR Edge → 1D 거리 비교 (axis-aligned absolute distance) — 극한의 단순함
  • CURVE Edge → SAT (Separating Axis Theorem) — 정밀한 2D OBB 충돌 판정
  • EdgeVehicleQueue → edge당 차량 리스트 정렬 — N² → N 축소의 비밀
  • 3-Zone Sensor → APPROACH / BRAKE / STOP 3단계 감속 체계

2. SAT (Separating Axis Theorem) 설명

2.1 SAT란 무엇인가

Separating Axis Theorem은 두 볼록 다각형(convex polygon)의 겹침을 판정하는 수학적 정리입니다. 핵심 원리는 놀라울 만큼 단순합니다:

SAT 핵심 원리

두 볼록 다각형이 겹치지 않는다면, 반드시 하나 이상의 분리축(separating axis)이 존재하여 두 도형의 투영(projection)이 겹치지 않는다.

역으로: 모든 후보 축에서 투영이 겹치면 → 두 도형은 반드시 충돌 중이다.

2.2 축 선택: 왜 변의 법선인가

볼록 다각형에서 분리축 후보는 각 변에 수직인 법선(normal)입니다. 사각형(Quad)의 경우 4변이므로:

사각형 A: 4변 → 4개 법선축 사각형 B: 4변 → 4개 법선축 ───────────────────────── 총 후보축: 8개 BUT: early exit — 하나라도 분리되면 즉시 "충돌 아님" 반환 실제 평균 체크 횟수: 1~3축 (대부분 첫 축에서 걸러짐)

2.3 투영(Projection) 개념

각 축에 대해 두 사각형의 모든 꼭짓점을 1차원 직선 위에 투영합니다. 투영 결과의 [min, max] 범위가 겹치는지 확인합니다.

축 (axis) ↓ ───────[====A====]──────────────── 투영 범위 A (minA ~ maxA) ────────────────[====B====]─────── 투영 범위 B (minB ~ maxB) ^^^^^^^^^^ 겹침 구간 존재 → 이 축에서는 분리 불가 vs. ───────[====A====]──────────────── 투영 범위 A ──────────────────────[====B====]─ 투영 범위 B ^ maxA < minB → 분리축 발견! 충돌 아님!

2.4 satQuadCheck 구현 상세

satQuadCheck()는 한 쪽 사각형의 4개 변 법선에 대해 양쪽 사각형을 투영하는 함수입니다. 두 번 호출해야 완전한 SAT 검사가 됩니다 (A의 축 4개 + B의 축 4개).

satQuadCheck 알고리즘 (4축 검사)

function satQuadCheck(data, baseA, baseB, idxA, idxB):
  for i = 0 to 3:                          // 4개 변 순회
    p1 = vertex[i],  p2 = vertex[(i+1) % 4]

    // 변의 법선벡터 계산 (90도 회전)
    axisX = -(p2.y - p1.y)
    axisY =  (p2.x - p1.x)

    // 축 길이가 거의 0이면 스킵 (축퇴 변)
    if (axisX² + axisY² < 1e-10) continue

    // 양쪽 사각형의 투영 범위 계산
    [minA, maxA] = projectQuad(A, axis)
    [minB, maxB] = projectQuad(B, axis)

    // 분리축 발견? → 겹침 없음!
    if (maxA < minB || maxB < minA):
      return false   // EARLY EXIT

  return true  // 4축 모두 겹침 → 이 쪽에서는 충돌

완전한 SAT 충돌 판정 (sensorCollision.ts 라인 63~66)

// 양쪽 모두 통과해야 최종 충돌
if (
  satQuadCheck(data, sensorBase, targetBase, SENSOR_QUAD_IDX, BODY_QUAD_IDX) &&
  satQuadCheck(data, targetBase, sensorBase, BODY_QUAD_IDX, SENSOR_QUAD_IDX)
) {
  return zone;  // 충돌 확인!
}

첫 번째 호출: Sensor Quad의 4축으로 검사 / 두 번째 호출: Body Quad의 4축으로 검사

3. 1D 레일 최적화 (핵심 혁신)

3.1 왜 가능한가: 레일 구속 조건

AMHS 시뮬레이터의 차량은 자유 공간이 아닌 레일(edge) 위에서만 이동합니다. 이 구속 조건이 극적인 최적화를 가능하게 합니다:

레일 구속이 만드는 기적

  • 일반 2D 시뮬레이터: 모든 차량 쌍을 비교 → O(N²) 또는 공간 분할(quadtree 등) 필요
  • VPS 레일 시뮬레이터: 차량은 같은 edge 또는 인접 edge에서만 충돌 가능
  • 결과: edge당 차량 수 = ~5-20대 → 비교 대상이 자연스럽게 제한됨
일반 2D 시뮬레이터 (N = 100,000) 모든 쌍 비교: N × (N-1) / 2 = ~5,000,000,000 비교/프레임 Quadtree: ~O(N log N) = ~1,700,000 비교/프레임 VPS 레일 시뮬레이터 (N = 100,000) edge 수: ~10,000 edges edge당 차량: ~10대 평균 edge당 비교: ~10 (인접 쌍) ────────────────────────── 총 비교 횟수: ~10,000 × 10 = ~100,000 비교/프레임 차량당: ~1 비교/vehicle = O(1)/vehicle!

3.2 EdgeVehicleQueue: 핵심 데이터 구조

EdgeVehicleQueue는 각 edge에 현재 위치한 차량 ID를 ratio(진행률) 순서로 정렬 관리하는 자료구조입니다.

메모리 레이아웃 (Int32Array 단일 버퍼)

EDGE_LIST_SIZE = 101  (1 count + 100 vehicle slots)

전체 버퍼: Int32Array[edgeCount × EDGE_LIST_SIZE]

Edge #1 (offset 0):
  [count=3] [veh_42] [veh_17] [veh_88] [0] [0] ... [0]
   ^offset   ^+1      ^+2      ^+3     (미사용 슬롯)

Edge #2 (offset 101):
  [count=1] [veh_05] [0] [0] ... [0]

Edge #3 (offset 202):
  [count=0] [0] [0] [0] ... [0]   ← 차량 없는 edge (즉시 스킵)

3.3 1D LINEAR 충돌 판정

직선 edge에서는 차량이 하나의 축(X 또는 Y)을 따라 이동합니다. SAT 없이 절댓값 거리 비교만으로 충돌을 판정합니다:

1D 거리 계산 (verifyFollowingCollision.ts)

// edge.axis가 "x"면 X좌표, "y"면 Y좌표만 사용
const axisIdx = edge.axis === "y" ? MovementData.Y : MovementData.X;
const frontPos = vehicleArrayData[ptrFront + axisIdx];
const backPos  = vehicleArrayData[ptrBack  + axisIdx];

const distance = Math.abs(frontPos - backPos);

// 센서 프리셋의 zone 길이 + 차체 길이로 hitZone 판정
const stopDist     = preset.zones.stop.leftLength     + vehicleLength;
const brakeDist    = preset.zones.brake.leftLength    + vehicleLength;
const approachDist = preset.zones.approach.leftLength + vehicleLength;
LINEAR 1D 충돌 판정 시각화 edge (직선 레일, axis = "x") ================================================================→ x [veh_B]----distance----[veh_A] 후행차 선행차 |<--- approachDist --->| (4.5m + bodyLen) | |<-- brakeDist -->| (1.2m + bodyLen) | | |<- stopDist >| (0.3m + bodyLen) distance ≤ stopDist → STOP (즉시 정지) distance ≤ brakeDist → BRAKE (강한 감속) distance ≤ approachDist → APPROACH (약한 감속) distance > approachDist → NONE (정상 주행)

4. 3-Zone 센서 시스템

4.1 3단계 감속 체계

VPS는 현실 세계의 차량 센서를 모방하여 3단계 Zone으로 점진적 감속을 구현합니다. 급정거를 방지하고 자연스러운 감속 곡선을 만듭니다.

ZoneIndex감속목적동작
APPROACH 0 -3 m/s² (STRAIGHT 기준) 먼 거리, 예비 감속 속도 > approachMinSpeed면 감속 적용
BRAKE 1 -4 m/s² (STRAIGHT 기준) 중간 거리, 적극 감속 속도 > brakeMinSpeed면 감속 적용
STOP 2 -∞ (즉시 정지) 가까운 거리, 비상 정지 VELOCITY=0, MOVING_STATUS=STOPPED

4.2 센서 포인트 메모리 구조

각 zone은 6개 포인트로 사각형 영역을 정의합니다. 포인트당 (x, y) 2개 float → zone당 12 floats.

SensorPoint 오프셋 (12 floats per zone)

SensorPoint = {
  FL_X: 0,  FL_Y: 1,    // Front-Left
  FR_X: 2,  FR_Y: 3,    // Front-Right
  BL_X: 4,  BL_Y: 5,    // Back-Left
  BR_X: 6,  BR_Y: 7,    // Back-Right
  SL_X: 8,  SL_Y: 9,    // Side-Left  (센서 끝점)
  SR_X: 10, SR_Y: 11,   // Side-Right (센서 끝점)
}

SENSOR_POINT_SIZE = 12 floats/zone
SENSOR_ZONE_COUNT = 3  (approach, brake, stop)
SENSOR_DATA_SIZE  = 3 × 12 = 36 floats/vehicle
영역사각형 구성 (반시계방향)용도
Sensor QuadFL → SL → SR → FR센서 감지 영역 (zone별로 크기 다름)
Body QuadFL → BL → BR → FR차량 차체 영역 (충돌 대상)

4.3 차량 + 3 Zone 시각화

APPROACH Zone (가장 큰 영역) +───────────────────────────────────+ / BRAKE Zone \ / +─────────────────────────────+ \ / / STOP Zone \ \ / / +───────────────────────+ \ \ / / / \ \ \ SL ───────/ / / 차량 진행 방향 → \ \ \──────── SR | | | | | | | +─────────────────────────+ | | | | | | 차량 BODY | | | | | | | BL ─────────────── BR | | | | SL ───────+ | | | | | | | +──────── SR \| | | Vehicle Body | | | / / \ | | | | | / / \| FL ─────────────── FR | | / / +─────────────────────────+ | / / \ | / / +─────────────────────────+/ / \ / +─────────────────────────────+ SAT 검사 시: Sensor Quad (FL-SL-SR-FR) vs Target의 Body Quad (FL-BL-BR-FR) Zone 2(STOP) → Zone 1(BRAKE) → Zone 0(APPROACH) 순서로 검사 가장 안쪽(STOP)부터 검사하여 가장 긴급한 zone을 우선 반환

4.4 Zone 검사 순서: inner → outer

checkSensorCollision() 핵심 로직

// check inner -> outer to prioritize strongest braking
for (let zone = 2; zone >= 0; zone--) {
  // zone 2 = STOP (가장 안쪽, 가장 긴급)
  // zone 1 = BRAKE
  // zone 0 = APPROACH (가장 바깥쪽)

  if (satQuadCheck(...sensorQuad...) && satQuadCheck(...bodyQuad...)) {
    return zone;  // 가장 안쪽에서 먼저 발견되면 즉시 반환
  }
}
return -1;  // 어떤 zone에도 충돌 없음

STOP zone에 동시 진입하면 APPROACH는 검사하지 않습니다. 가장 긴급한 응답이 보장됩니다.

6. 충돌 체크 3가지 유형

6.1 Following Collision (같은 edge 내 후행 충돌)

verifyFollowingCollision() — 같은 edge 위에서 인접 차량 쌍을 순서대로 비교합니다.

Edge #17 (ratio 정렬된 EdgeVehicleQueue) ================================================================→ [count=4] [veh_42, ratio=0.1] [veh_17, ratio=0.3] [veh_88, ratio=0.6] [veh_05, ratio=0.9] ^ ^ ^ ^ | | | | +-------비교---------+ | | (front=42, back=17) | | +-------비교--------+ | (front=17, back=88) | +-------비교--------+ (front=88, back=05) 총 비교 횟수: count - 1 = 3회 (4대 차량에 대해) 각 비교는 LINEAR이면 1D 거리, CURVE면 SAT

Following Collision 알고리즘

for (i = 1; i < count; i++):
  frontVehId = queue[offset + 1 + (i-1)]   // 앞차
  backVehId  = queue[offset + 1 + i]        // 뒷차

  if (!shouldCheckCollision(backVehId, delta, timers, 33ms)):
    continue   // 타이머 미경과 → 스킵

  if (edge.vos_rail_type):          // CURVE
    hitZone = checkSensorCollision(sensorPoints, backVeh, frontVeh)
  else:                              // LINEAR
    distance = |frontPos - backPos|
    hitZone = determineLinearHitZone(distance, stop, brake, approach)

  applyCollisionZoneLogic(hitZone, data, ptrBack, frontVehId)

6.2 NextPath Collision (다음 edge 후미 충돌)

verifyNextPathCollision() — edge 선두 차량이 다음 edge의 후미 차량과 충돌하는지 검사합니다. 짧은 edge를 관통하여 연쇄 탐색합니다.

NextPath Collision 시나리오 Edge A Edge B (다음 edge) Edge C (그 다음) =====================→ =====================→ =====================→ [veh_42] [veh_17] [veh_88] [veh_05] ^ ^ | 선두 차량 | 후미 차량 +--- 비교 --------+ veh_17 (Edge A 선두) vs veh_88 (Edge B 후미) 짧은 edge 관통 탐색 (SHORT_LINEAR_THRESHOLD = 2m) Edge A Edge B (1.5m, 짧음!) Edge C =========→ ====→ ==============→ [veh_17] (비어있음) [veh_05] ^ ^ | Edge B가 짧고 비어있음 → | +------------ 관통하여 Edge C까지 탐색 ---------+ veh_17 (Edge A 선두) vs veh_05 (Edge C 후미)

짧은 Edge 관통 로직 (BFS)

const checkedEdges = new Set();
const edgesToCheck = [...nextEdges];

while (edgesToCheck.length > 0 && mostCriticalHitZone !== STOP):
  nextEdgeIdx = edgesToCheck.shift()
  if (checkedEdges.has(nextEdgeIdx)) continue
  checkedEdges.add(nextEdgeIdx)

  // 후미 차량과 충돌 체크
  result = checkEdgeTailCollision(nextEdgeIdx, myVehIdx, ...)

  // 짧은 LINEAR edge면 그 다음 edge도 큐에 추가
  if (nextEdge.type === LINEAR && nextEdge.distance < 2m):
    edgesToCheck.push(...nextEdge.nextEdgeIndices)

짧은 edge가 연속되면 여러 edge를 관통하여 실제 충돌 대상을 찾습니다.

7. Rough Distance Filter

SAT 연산은 8축 투영이 필요하므로 비용이 있습니다. roughDistanceCheck()는 SAT 전에 빠른 거리 필터로 ~90% 후보를 제거합니다.

roughDistanceCheck 구현

function roughDistanceCheck(sensorPointArray, vehIdx1, vehIdx2, threshold):
  // FL(Front-Left) 포인트 기반 유클리드 거리² 비교
  dx = data[base1 + FL_X] - data[base2 + FL_X]
  dy = data[base1 + FL_Y] - data[base2 + FL_Y]

  return dx*dx + dy*dy ≤ threshold*threshold
  //     ^^^^^^^^^^^^^^^^^^
  //     sqrt 없이 제곱 비교 (성능 최적화)

8. 충돌 응답 (applyCollisionZoneLogic)

applyCollisionZoneLogic()은 감지된 hitZone에 따라 차량의 속도, 이동 상태, 감속도를 즉시 반영합니다.

applyCollisionZoneLogic 핵심 코드

if (hitZone === HitZone.NONE):
  data[ptr + COLLISION_TARGET] = -1
  data[ptr + DECELERATION] = 0
  if (MOVING_STATUS === STOPPED):
    MOVING_STATUS = MOVING       // 정지 해제 → 재출발
  return

if (hitZone === HitZone.STOP):
  MOVING_STATUS = STOPPED
  VELOCITY = 0                   // 즉시 정지
  DECELERATION = 0
  return

// BRAKE 또는 APPROACH
const minSpeed = isBrake ? brakeMinSpeed : approachMinSpeed
const decValue = isBrake ? preset.zones.brake.dec : preset.zones.approach.dec

if (velocity > minSpeed):
  DECELERATION = decValue        // 감속 적용
else:
  DECELERATION = 0               // 최소 속도 이하면 감속 중지

최소 속도 보호

BRAKE와 APPROACH zone에서는 속도가 minSpeed 이하로 떨어지면 감속을 중지합니다. 이는 차량이 영원히 가속/감속을 반복하는 oscillation을 방지합니다.

  • approachMinSpeed: config에서 설정 (기본 ~2 m/s)
  • brakeMinSpeed: config에서 설정 (기본 ~1 m/s)

9.1 Throttling (차량별 충돌 체크 주기)

33ms 간격 Throttling

물리 엔진은 16ms (60fps) 간격으로 동작하지만, 충돌 체크는 차량별로 33ms (~30fps) 간격으로 수행합니다.

shouldCheckCollision(vehId, delta, collisionCheckTimers, checkInterval=33ms)

elapsed = (이전 누적 시간) + delta * 1000
if (elapsed < 33ms):
  return false     // 스킵 (이전 HIT_ZONE 유지)
else:
  타이머 리셋
  return true      // 체크 수행

효과: 충돌 체크 연산량 ~50% 감소. hitZone은 33ms 동안 유지되므로 시각적/물리적 차이 없음.
