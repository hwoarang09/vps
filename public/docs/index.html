VPS (Virtual Physics Simulator)는 100,000대 이상의 차량을 실시간으로 시뮬레이션하는 고성능 AMHS(Automated Material Handling System) 시뮬레이터입니다.

Core Architecture

Main Thread (React + Three.js) ├─ UI 렌더링, 사용자 입력 ├─ Three.js InstancedMesh (vehicle 위치 읽기) └─ SharedArrayBuffer READ ONLY │ │ zero-copy (Float32Array views) │ Worker Thread(s) (Simulation Engine) ├─ 물리 계산, 충돌 검출, 경로 탐색 ├─ Lock 관리, Transfer 주문 처리 └─ SharedArrayBuffer WRITE │ │ OPFS sync write (binary) │ OPFS / MQTT └─ 고빈도 로그 저장 → Python 분석

Key Design Principles

  • Thread Separation — 시뮬레이션(Worker)은 DOM/Three.js에 접근하지 않음. 렌더링(Main)은 물리 계산을 하지 않음.
  • Zero-Copy Communication — 차량 위치 데이터는 SharedArrayBuffer를 통해 전달. postMessage 없이 직접 메모리 읽기.
  • Per-Fab Isolation — 각 Fab(공장)은 독립된 시뮬레이션 컨텍스트. 서로 직접 상호작용하지 않음.
  • Zero-GC Hot Path — 충돌/이동/Dijkstra 등 매 프레임 실행 코드는 사전 할당된 버퍼만 사용.
  • Data-Driven Config — 모든 물리/라우팅/Lock 파라미터를 런타임에 Fab별로 변경 가능.
