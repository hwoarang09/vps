MovementData FieldsOffset 0 ~ 17 — 차량 운동 상태의 18개 필드 상세

MovementData는 Vehicle Data Array의 처음 18개 필드(offset 0~17)로, 차량의 물리 위치, 속도, 경로 진행 상태를 나타냅니다. Worker 스레드의 updateMovement() 4단계 파이프라인이 매 프레임 갱신하고, Main Thread의 렌더러가 읽어 GPU에 전달합니다.

4단계 파이프라인 개요

Phase 1 vehiclePhysics.ts 가속도/충돌/속도 계산 → newVelocity, rawNewRatio Phase 2 vehicleTransition.ts Edge 전환 판정 → finalEdgeIndex, finalRatio Phase 3 vehiclePosition.ts 좌표 보간 → finalX, finalY, finalZ, finalRotation Phase 4 vehicleState.ts Float32Array 기록 → X, Y, Z, ROTATION, VELOCITY, EDGE_RATIO, CURRENT_EDGE
위치 / 자세 (Offset 0 ~ 3)

0 X Float32

용도차량의 월드 좌표계 X 위치. Edge 위의 ratio를 from_node/to_node 좌표로 보간하여 산출된 값이다.
Writer (쓰기) vehicleState.ts — commitVehicleState() — Phase 4에서 매 프레임 기록.
vehiclePosition.ts — updateVehiclePosition()이 interpolatePositionTo()로 계산한 결과를 전달.
Reader (읽기) FabContext/render.ts — writeToRenderRegion() — 렌더 버퍼에 fabOffset 적용 후 복사.
collisionCommon.ts — 센서 충돌 검사 시 차량 위치 참조.
edgeTransition.ts — checkAndReportUnusualMove() — UnusualMove 이벤트에 좌표 포함.
값 범위맵 좌표 범위 (실수, 단위: m). Edge의 from_node.x ~ to_node.x 보간값.
관련 문서Three.js Renderer — GPU InstancedMesh 업데이트

1 Y Float32

용도차량의 월드 좌표계 Y 위치. Three.js 좌표계에서 수평면의 다른 축(Z-up 맵에서는 depth)에 해당한다.
Writer (쓰기)vehicleState.ts — commitVehicleState() — Phase 4에서 매 프레임 기록.
Reader (읽기) FabContext/render.ts — 렌더 버퍼 복사 (fabOffsetY 적용).
collisionCommon.ts — 센서 충돌 검사.
값 범위맵 좌표 범위 (실수, 단위: m).
관련 문서Three.js Renderer

2 Z Float32

용도차량의 월드 좌표계 Z 위치(고도). Edge 보간값에 vehicleZOffset을 더하여 차량이 레일 위에 떠 있도록 조정한다.
Writer (쓰기)vehicleState.ts — commitVehicleState() — Phase 4에서 기록. interpolatePositionTo()가 config.vehicleZOffset을 합산.
Reader (읽기)FabContext/render.ts — 렌더 버퍼 복사 (Z는 fabOffset 미적용).
값 범위맵 좌표 + vehicleZOffset (실수, 단위: m). 일반적으로 0에 가까운 값.
관련 문서Three.js Renderer

3 ROTATION Float32

용도차량의 Y축 회전각(radians). Edge의 from_node에서 to_node를 바라보는 방향을 atan2로 계산한다. 곡선 Edge에서는 ratio에 따라 연속 보간된다.
Writer (쓰기)vehicleState.ts — commitVehicleState() — Phase 4에서 기록. positionInterpolator.ts — interpolatePositionTo()가 계산.
Reader (읽기) FabContext/render.ts — 렌더 버퍼 복사, InstancedMesh 회전에 사용.
sensorPoints.ts — updateSensorPoints() — 센서 영역을 차량 회전에 맞춰 변환.
값 범위-PI ~ +PI (radians). constants.ts 주석에는 "radians"로 명시.
관련 문서Three.js Renderer — InstancedMesh 회전 적용
속도 / 가감속 (Offset 4 ~ 6)

4 VELOCITY Float32

용도차량의 현재 속도(m/s). 매 프레임 물리 계산 결과로 갱신되며, Edge 위 이동 거리(ratio 증분) 산출의 핵심 입력이다.
Writer (쓰기) vehicleState.ts — commitVehicleState() — Phase 4에서 최종 속도 기록.
vehiclePhysics.ts — processEmergencyStop() — hitZone=STOP일 때 즉시 0으로 기록.
movementUpdate.ts — shouldSkipUpdate() — STOPPED/PAUSED 시 0으로 초기화.
lock-handlers.ts — handleLockWait() — Lock 대기 시 0으로 강제 설정.
Reader (읽기) vehiclePhysics.ts — calculateVehiclePhysics() — 다음 프레임 속도 계산의 입력.
IndividualControlPanel.tsx — UI 패널에서 차량 현재 속도 표시.
KpiHud.tsx — KPI 대시보드 속도 통계.
lock-handlers.ts — merge braking 잔여 거리 계산.
값 범위0.0 ~ linearMaxSpeed (m/s). 곡선에서는 curveMaxSpeed가 상한. 음수 없음.
관련 문서 Simulation Engine — 물리 루프 | Lock System — WAIT 시 강제 정지

5 ACCELERATION Float32

용도차량의 가속도(m/s2). 양수 값으로, 차량이 목표 속도에 도달할 때까지 적용되는 기본 가속률이다. 감속이 활성화되면 가속은 0으로 대체된다.
Writer (쓰기) VehicleDataArrayBase.ts — 차량 초기화 시 프리셋 가속도 값으로 설정.
IndividualControlPanel.tsx — UI에서 수동 제어 시 직접 기록.
Reader (읽기) vehiclePhysics.ts — calculateVehiclePhysics() — 매 프레임 읽어 calculateNextSpeed()에 전달.
speedCalculator.ts — calculateNextSpeed() — v_next = v + a * dt 계산.
값 범위>= 0 (m/s2, 양수). 일반적으로 0.5 ~ 3.0 범위.
관련 문서 Simulation Engine | Parameter Preset — 가속도 프리셋 관리

6 DECELERATION Float32

용도차량의 감속도(m/s2). 센서 충돌 감지, 곡선 사전 감속, merge 사전 감속 시 음수 값으로 설정된다. -Infinity는 즉시 정지(hard stop)를 의미한다.
Writer (쓰기) VehicleDataArrayBase.ts — 차량 초기화 시 기본 감속도 설정.
vehiclePhysics.ts — processEmergencyStop() — 긴급 정지 시 0으로 리셋.
sensorCollision 모듈 — HitZone 판정에 따라 감속도 기록.
Reader (읽기) vehiclePhysics.ts — calculateVehiclePhysics() — 충돌 감속, 곡선 감속, merge 감속 중 최대값 선택.
speedCalculator.ts — calculateNextSpeed() — -Infinity 체크 후 속도 계산.
값 범위<= 0 (m/s2, 음수 또는 0). 특수값: -Infinity = hard stop.
관련 문서Simulation Engine — 3가지 감속 요소 통합 로직
Edge 진행 상태 (Offset 7 ~ 9)

7 EDGE_RATIO Float32

용도현재 Edge 위에서의 진행률. 0.0은 Edge 시작점(from_node), 1.0은 끝점(to_node)이다. 좌표 보간과 Edge 전환 판정의 핵심 값.
Writer (쓰기) vehicleState.ts — commitVehicleState() — Phase 4에서 최종 ratio 기록.
edgeTransition.ts — handleEdgeTransition() — Edge 전환 시 overflow를 다음 Edge의 ratio로 변환하여 store.moveVehicleToEdge()에 전달.
Reader (읽기) vehiclePhysics.ts — rawNewRatio = edgeRatio + (v * dt) / distance 계산.
checkpoint-processor.ts — 현재 ratio와 checkpoint ratio 비교하여 도달 판정.
lock-handlers.ts — calcRemainingDistToMerge() — merge까지 잔여 거리 계산.
curveBraking.ts / mergeBraking.ts — 사전 감속 거리 계산.
값 범위0.0 ~ 1.0. Phase 1에서 rawNewRatio > 1.0이 될 수 있으나, Edge 전환 후 새 Edge 기준으로 재계산되어 최종 기록값은 0~1 범위.
관련 문서 Simulation Engine — ratio 기반 이동 모델 | Lock System — checkpoint 도달 판정

8 MOVING_STATUS Float32

용도차량의 이동 상태를 나타내는 enum 값. 파이프라인 진입 여부를 결정하는 게이트 역할을 한다.
Writer (쓰기) vehicleTransition.ts — processSameEdgeLogic() — target ratio 도달 시 STOPPED로 전환.
lock-handlers.ts — handleLockWait() — Lock blocked 시 STOPPED, grant 시 MOVING으로 전환.
TransferMgr — 새 경로 할당 시 MOVING으로 설정.
IndividualControlPanel.tsx — UI에서 수동 제어 시 직접 변경.
Reader (읽기) movementUpdate.ts — shouldSkipUpdate() — STOPPED/PAUSED이면 파이프라인 스킵.
checkpoint-processor.ts — 차량 상태 확인.
EdgeVehicleQueue.ts — Edge 위 차량 상태 조회.
값 범위 enum MovingStatus:
0 = STOPPED — 정지 (velocity 강제 0)
1 = MOVING — 이동 중 (파이프라인 실행)
2 = PAUSED — 일시정지 (파이프라인 스킵, velocity 유지)
관련 문서 Simulation Engine | Lock System — WAIT/GRANT 상태 전환

9 CURRENT_EDGE Float32

용도차량이 현재 위치한 Edge의 인덱스(1-based). Edge 배열 접근 시 edgeArray[CURRENT_EDGE - 1]로 사용한다. 0은 무효 sentinel 값.
Writer (쓰기) vehicleState.ts — commitVehicleState() — Phase 4에서 finalEdgeIndex 기록.
edgeTransition.ts — store.moveVehicleToEdge() — Edge 전환 시 내부적으로 갱신.
Reader (읽기) vehiclePhysics.ts — 현재 Edge 참조하여 distance, rail_type 확인.
lock-handlers.ts — 현재 Edge의 to_node로 merge node 판별, auto-release 체크.
checkpoint-processor.ts / checkpoint-loader.ts — 현재 Edge 기준 checkpoint 탐색.
TransferMgr — 경로 계산의 시작점으로 사용.
collisionCommon.ts — 동일 Edge 위 차량 간 거리 계산.
값 범위0 ~ edgeArray.length (정수, 1-based). 0 = 무효(차량 미배치).
관련 문서 Simulation Engine | Lock System | Routing
Next Edge 큐 (Offset 10 ~ 15)

NEXT_EDGE_0~4는 5칸짜리 선입선출(FIFO) 큐로, 차량이 다음에 진입할 Edge들을 미리 저장한다. Edge 전환 시 NEXT_EDGE_0이 소비되고 나머지가 한 칸씩 shift된다. Checkpoint의 MOVE_PREPARE 플래그가 pathBuffer에서 큐를 채운다.

10 NEXT_EDGE_0 Float32

용도바로 다음에 진입할 Edge의 인덱스(1-based). Edge 전환 시 이 값이 소비되어 CURRENT_EDGE가 된다.
Writer (쓰기) lock-handlers.ts — handleMovePrepare() — Checkpoint MOVE_PREPARE 시 pathBuffer에서 Edge를 읽어 채움.
edgeTransition.ts — shiftNextEdges() — Edge 전환 후 NEXT_EDGE_1 값으로 교체.
Reader (읽기) edgeTransition.ts — handleEdgeTransition() — 다음 Edge 존재 여부 및 전환 가능성 판정.
vehicleTransition.ts — checkAndTriggerTransfer() — EMPTY 상태에서 Transfer 큐 트리거.
값 범위0 ~ edgeArray.length (정수, 1-based). 0 = 비어 있음.
관련 문서Lock System — MOVE_PREPARE checkpoint

11 NEXT_EDGE_1   12 NEXT_EDGE_2   13 NEXT_EDGE_3   14 NEXT_EDGE_4 Float32 × 4

용도예약된 다음 Edge 큐의 2~5번째 슬롯. 곡선 사전 감속(curveBraking)에서 전방 Edge들의 rail_type을 미리 확인하는 데 사용된다.
Writer (쓰기) lock-handlers.ts — handleMovePrepare() — pathBuffer에서 최대 5개까지 채움 (targetEdge까지만).
edgeTransition.ts — shiftNextEdges() — 전환 시 한 칸씩 앞으로 shift, 마지막(NEXT_EDGE_4)은 0으로 비움.
Reader (읽기) curveBraking.ts — 전방 곡선 Edge 탐지 시 순회.
mergeBraking.ts — 전방 merge 노드 탐지.
값 범위0 ~ edgeArray.length (정수, 1-based). 0 = 비어 있음.
관련 문서 Simulation Engine — 곡선 사전 감속 | Lock System — checkpoint 기반 Edge 준비

15 NEXT_EDGE_STATE Float32

용도NEXT_EDGE_0의 준비 상태를 나타내는 enum. Edge 전환은 이 값이 READY일 때만 가능하다.
Writer (쓰기) vehicleTransition.ts — checkAndTriggerTransfer() — ratio >= 0이고 EMPTY일 때 PENDING으로 설정하고 Transfer 큐에 등록.
lock-handlers.ts — handleMovePrepare() — NEXT_EDGE_0이 채워지면 READY로, 비어 있으면 EMPTY로 설정.
edgeTransition.ts — shiftNextEdges() — shift 후 NEXT_EDGE_0이 0이면 EMPTY로 리셋.
Reader (읽기) edgeTransition.ts — checkCanTransitionToNextEdge() — READY가 아니면 전환 불가.
vehicleTransition.ts — processEdgeTransitionLogic() — 전환 조건에 READY 체크 포함.
값 범위 enum NextEdgeState:
0 = EMPTY — 다음 Edge 없음 (Transfer 필요)
1 = PENDING — Transfer 큐에 등록됨, 경로 계산 대기
2 = READY — 다음 Edge 준비 완료, 전환 가능
관련 문서Lock System — checkpoint MOVE_PREPARE 처리
보조 파라미터 (Offset 16 ~ 17)

16 TARGET_RATIO Float32

용도현재 Edge 위에서의 목표 정차 위치(ratio). 차량은 이 ratio에 도달하면 정지(STOPPED)한다. 일반적으로 1.0(Edge 끝까지 이동)이며, 스테이션 정차 등에서 중간값이 설정된다.
Writer (쓰기) edgeTransition.ts — handleEdgeTransition() — Edge 전환 시 새 Edge의 TARGET_RATIO를 1.0 또는 nextTargetRatio(경로 예약값)로 설정.
TransferMgr — 경로 할당 시 최종 Edge의 정차 ratio 설정.
Reader (읽기) vehiclePhysics.ts — calculateVehiclePhysics() — clampTargetRatio()로 0~1 범위 보정 후 전달.
vehicleTransition.ts — processEdgeTransitionLogic() — rawNewRatio >= 1 && targetRatio === 1일 때 Edge 전환 트리거.
vehicleTransition.ts — checkTargetReached() — 동일 Edge에서 target 도달 시 정차.
값 범위0.0 ~ 1.0. 내부에서 clampTargetRatio()로 범위 보정됨.
관련 문서Simulation Engine — 정차 로직

17 OFFSET Float32

용도레일 중심선으로부터의 횡방향 편차(lane offset). 다중 차선이나 추월 시나리오에서 차량이 레일 중심에서 벗어난 거리를 나타낸다.
Writer (쓰기) VehicleDataArrayBase.ts — 차량 초기화 시 기본값(보통 0) 설정.
Reader (읽기) verifyMergeCollision.ts — merge 충돌 검사 시 차량의 횡방향 위치 보정에 사용.
VehicleDataArrayBase.ts — getOffset() / setOffset() — getter/setter 인터페이스 제공.
값 범위실수 (단위: m). 일반적으로 0.0. 양수 = 우측, 음수 = 좌측 편차.
