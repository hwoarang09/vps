Movement Physics4-Phase Pipeline, Edge Transition, Position Interpolation, Zero-GC

1. 개요

VPS Movement 시스템은 매 시뮬레이션 프레임마다 4단계 파이프라인을 실행하여 차량의 물리 상태를 업데이트합니다. 모든 단계는 Zero-GC Scratchpad 패턴을 사용하여 GC 압력 없이 동작합니다.

Phase 1: calculateVehiclePhysics 충돌(hitZone) 반영 / 커브 사전 감속 / Merge 사전 감속 최종 가감속 결정 → calculateNextSpeed Phase 2: processVehicleTransition rawNewRatio >= 1 → Edge 전환 (overflow distance 기반) Multi-hop while loop / NEXT_EDGE shift pipeline TARGET_RATIO 정차 위치 / merge lock 해제 Phase 3: updateVehiclePosition LINEAR: 시작점→끝점 선형 보간 CURVE: renderingPoints[] 세그먼트 보간 calculateStableVector: 최소 거리 임계값 look-ahead/back Phase 4: commitVehicleState SharedArrayBuffer에 최종 값 기록 updateSensorPoints() → 다음 프레임 충돌 검출용

Entry: src/common/vehicle/movement/movementUpdate.ts — updateMovement()

파이프라인 진입 조건

shouldSkipUpdate()가 MovingStatus를 확인합니다. PAUSED이면 즉시 스킵, STOPPED이면 velocity를 0으로 설정 후 스킵합니다. 오직 MOVING 상태의 차량만 4-phase 파이프라인을 통과합니다.

파이프라인 호출 순서

for (let i = 0; i < actualNumVehicles; i++) {
  const ptr = i * VEHICLE_DATA_SIZE;
  if (shouldSkipUpdate(data, ptr)) continue;

  // Phase 1: 물리 계산
  calculateVehiclePhysics(ctx, i, data, ptr, SCRATCH_PHYSICS);
  if (SCRATCH_PHYSICS.shouldSkip) continue;

  // Phase 2: Edge 전환
  processVehicleTransition(ctx, i, data, ptr,
    SCRATCH_PHYSICS, SCRATCH_VEHICLE_TRANSITION);

  // Phase 3: 위치 보간
  updateVehiclePosition(ctx, i, data, ptr,
    SCRATCH_VEHICLE_TRANSITION, SCRATCH_VEHICLE_POSITION);

  // Phase 4: 최종 기록
  commitVehicleState(ctx, i, data, ptr,
    SCRATCH_VEHICLE_TRANSITION.finalEdgeIndex,
    SCRATCH_VEHICLE_POSITION);
}

2. Phase 1: 물리 계산 (calculateVehiclePhysics)

차량의 가속도, 충돌 감지 결과, 사전 감속 여부를 종합하여 새 속도(newVelocity)와 새 Edge 비율(rawNewRatio)을 계산합니다.

File: src/common/vehicle/movement/vehiclePhysics.ts

2.1 처리 순서

1. calculateHitZone hitZone 2 → processEmergencyStop → velocity=0, shouldSkip=true hitZone 0/1 → 접근/제동 감속으로 변환 2. checkCurvePreBraking 직선 Edge에서 전방 곡선까지 거리 계산 brakeDistance = (v² - vTarget²) / (2|a|) checkpoint 통과 시 감속 시작 3. checkMergePreBraking LOCK_WAIT checkpoint 까지 잔여 거리 계산 requiredDecel = -(v² - vTarget²) / (2s) 4. decideFinalAcceleration sensorDecel / curveDecel / mergeDecel 중 최대 선택 maxDecel > 0 → accel=0, decel=-maxDecel 5. calculateNextSpeed v_next = v + a * dt (maxSpeed clamp)

2.2 충돌 hitZone 반영

hitZone의미동작
-1충돌 없음정상 가속
0Approach zone접근 감속 (deceleration 값 사용)
1Brake zone제동 감속 (deceleration 값 사용)
2Stop zone긴급 정지 — velocity=0, SENSORED flag, 이후 Phase 스킵

긴급 정지 (hitZone === 2)

processEmergencyStop()은 즉시 VELOCITY=0, DECELERATION=0으로 설정하고 StopReason.SENSORED 비트를 켭니다. shouldSkip=true를 반환하여 Phase 2~4를 건너뜁니다.

2.3 커브 사전 감속 (Curve Pre-Braking)

직선 Edge 위의 차량이 전방 곡선에 진입하기 전에 curveMaxSpeed까지 속도를 줄이는 메커니즘입니다.

File: src/common/vehicle/movement/curveBraking.ts

Checkpoint 기반 감속 판단

  1. transferMgr.findDistanceToNextCurve()로 전방 곡선까지 거리(distanceToCurve) 계산
  2. brakeDistance = (v² - curveMaxSpeed²) / (2|preBrakeDecel|) 계산
  3. checkpointDistance = distanceToCurve - brakeDistance
  4. checkpointDistance <= 0이면 감속 시작 — transferMgr.startCurveBraking()
파라미터기본값설명
linearPreBrakeDeceleration-2 m/s²직선 구간에서 적용할 감속도
curvePreBrakeCheckInterval100 ms체크 주기 (-1이면 매 프레임)
curveMaxSpeedconfig 설정곡선 구간 목표 속도

상태 머신

IDLE ─── checkpointDistance <= 0 ──→ BRAKING │ v <= curveMaxSpeed ───→ clearCurveBrakeState → IDLE │ 곡선 Edge 진입 ────────→ clearCurveBrakeState → IDLE

2.4 Merge 사전 감속 (Merge Pre-Braking)

합류점(merge node) 앞 LOCK_WAIT checkpoint에 다가가는 차량이 정지할 수 있도록 감속합니다.

File: src/common/vehicle/movement/mergeBraking.ts

감속 조건

  1. CURRENT_CP_FLAGS에 LOCK_WAIT 비트가 설정됨
  2. Deadlock zone merge가 아님 (gate가 별도 stop 처리)
  3. WAIT CP까지 잔여 거리 > 0
  4. requiredDecel <= preBrakeDecel (여유 없음)

거리 계산 (distanceToCp)

// Same edge
dist = (cpRatio - currentRatio) * currentEdge.distance

// Multi-edge (NEXT_EDGE_0~4 traversal)
dist = (1 - currentRatio) * currentEdge.distance
     + sum(중간 edge.distance)
     + cpRatio * cpEdge.distance

필요 감속도 공식

// holder 아닌 경우: vTarget = 0 (정지)
// holder인 경우: vTarget = curveMaxSpeed (느리게 통과)
requiredDecel = -(v² - vTarget²) / (2 * distToWait)

2.5 최종 가감속 결정 (decideFinalAcceleration)

세 가지 감속 요소를 비교하여 가장 큰 감속도(max deceleration wins)를 선택합니다.

감속 요소소스트리거
sensorDecelhitZone >= 0충돌 센서 감지
curveDecelcurveBrakeResult전방 곡선 checkpoint 통과
mergeDecelmergeBrakeResultLOCK_WAIT CP 접근
const maxDecel = Math.max(sensorDecel, curveDecel, mergeDecel);
// maxDecel > 0 → accel = 0, decel = -maxDecel (감속 적용)
// maxDecel = 0 → accel = baseAcceleration, decel = 0 (가속 적용)

곡선 가속도 분리

곡선 Edge에서는 baseAcceleration 대신 config.curveAcceleration을 사용합니다. 직선과 곡선의 가속 특성을 독립 설정할 수 있습니다.

3. Phase 2: Edge Transition (processVehicleTransition)

Phase 1에서 계산된 rawNewRatio가 1 이상이면 현재 Edge를 벗어난 것이므로 다음 Edge로 전환합니다.

File: src/common/vehicle/movement/vehicleTransition.ts, edgeTransition.ts

3.1 전환 조건

const shouldTransition =
  rawNewRatio >= 1 &&
  (targetRatio === 1 || nextEdgeState === NextEdgeState.READY);

Transfer 큐 트리거

ratio >= 0이고 NEXT_EDGE_STATE === EMPTY이면 상태를 PENDING으로 변경하고 transferMgr.enqueueVehicleTransfer()를 호출합니다. 이것이 다음 Edge 결정의 시작점입니다.

3.2 Overflow Distance 계산

Edge를 넘어간 거리(overflow)를 다음 Edge의 ratio로 변환합니다.

overflowDist = (currentRatio - 1) * currentEdge.distance
nextRatio    = overflowDist / nextEdge.distance

3.3 Multi-hop While Loop

높은 속도에서 한 프레임에 여러 짧은 Edge를 연속 통과할 수 있습니다. handleEdgeTransition은 while (currentRatio >= 1) 루프로 이를 처리합니다.

handleEdgeTransition while (currentRatio >= 1) { overflowDist = (ratio - 1) * edge.distance canTransition? → NEXT_EDGE_STATE === READY && nextEdgeIndex > 0 ├─ No → ratio = 1, break (Edge 끝에서 정지) └─ Yes → store.moveVehicleToEdge(vehIdx, nextEdgeIdx, newRatio) updateSensorPresetForEdge() // STRAIGHT/CURVE_LEFT/RIGHT/U_TURN shiftNextEdges() // NEXT_EDGE_0~4 shift currentEdge = nextEdge currentRatio = overflowDist / nextEdge.distance }

3.4 NEXT_EDGE Shift Pipeline

Edge 전환 시 NEXT_EDGE_0~4 배열이 한 칸씩 shift됩니다. 마지막 슬롯은 0(빈 값)으로 채워집니다.

NEXT_EDGE_0 ← NEXT_EDGE_1
NEXT_EDGE_1 ← NEXT_EDGE_2
NEXT_EDGE_2 ← NEXT_EDGE_3
NEXT_EDGE_3 ← NEXT_EDGE_4
NEXT_EDGE_4 ← 0  // 마지막은 비움

// NEXT_EDGE_0 === 0 이면 STATE = EMPTY (refill 트리거)

PathBuffer 동기화

pathBufferFromAutoMgr (Int32Array)도 동일하게 shift합니다. Refill은 checkpoint에서 관리되며, transition 시점에서는 shift만 수행합니다.

3.5 NextEdgeState Machine

상태값의미전이
EMPTY초기/비움다음 Edge 미결정ratio >= 0 시 PENDING으로
PENDING대기중TransferMgr가 처리 중경로 결정 완료 시 READY로
READY준비됨다음 Edge 확정전환 완료 후 shift → EMPTY 가능

3.6 TARGET_RATIO: 정차 위치

Edge 내 특정 비율에서 정차해야 하는 경우 (station 도착, lock 대기 등) TARGET_RATIO를 사용합니다.

동일 Edge Target 도달

Edge 전환이 발생하지 않고 rawNewRatio >= targetRatio이면:

  • finalRatio = targetRatio (초과분 무시)
  • finalVelocity = 0
  • MovingStatus = STOPPED

Edge 전환이 발생하면 momentum은 유지됩니다 (target limit 미적용).

3.7 Merge Lock 해제

// Edge 전환 발생 시 (finalEdgeIndex !== currentEdgeIndex)
if (lockMgr.isMergeNode(prevToNode)) {
  lockMgr.releaseLock(prevToNode, vehId);
}

이전 Edge의 to_node가 merge node이면, 해당 노드의 Lock을 해제합니다.

3.8 UnusualMove 감지

Edge 전환 시 currentEdge.to_node !== nextEdge.from_node이면 연결되지 않은 Edge로의 이동(워프)이 감지되어 onUnusualMove 콜백이 호출됩니다.

4. Phase 3: Position Interpolation (updateVehiclePosition)

Edge ratio를 실제 월드 좌표(x, y, z)와 rotation으로 변환합니다.

File: src/common/vehicle/movement/vehiclePosition.ts, positionInterpolator.ts

4.1 LINEAR 보간

시작점과 끝점 사이를 ratio로 선형 보간합니다.

x = pStart.x + (pEnd.x - pStart.x) * ratio
y = pStart.y + (pEnd.y - pStart.y) * ratio
z = vehicleZOffset  (기본 0.15m)

Rotation (4방향)

LINEAR Edge는 주축 기반 4방향 회전을 사용합니다.

조건rotation
|dx| >= |dy| && dx >= 00° (오른쪽)
|dx| >= |dy| && dx < 0180° (왼쪽)
|dx| < |dy| && dy >= 090° (위)
|dx| < |dy| && dy < 0-90° (아래)

4.2 CURVE 보간

곡선 Edge는 renderingPoints[] 배열의 세그먼트 사이를 보간합니다.

safeRatio  = clamp(ratio, 0, 1)
floatIndex = safeRatio * (points.length - 1)
index      = floor(floatIndex)
segRatio   = floatIndex - index

x = p1.x + (p2.x - p1.x) * segRatio
y = p1.y + (p2.y - p1.y) * segRatio

4.3 calculateStableVector

곡선의 rotation 계산은 인접 포인트 간 거리가 너무 짧으면 불안정해질 수 있습니다. calculateStableVector는 최소 거리 임계값 MIN_DIST_SQ = 0.01 (0.1m²)을 적용합니다.

calculateStableVector dx = p2.x - p1.x, dy = p2.y - p1.y distSq = dx*dx + dy*dy if (distSq < 0.01) { Look-ahead: index+2, +3, ... → maxIndex 까지 탐색 if (여전히 짧음) { Look-back: index-1, -2, ... → 0 까지 역방향 탐색 } } rotation = atan2(dy, dx) * (180/PI) rotation = ((rotation % 360) + 360) % 360

Fallback

renderingPoints가 비어있으면 x=0, y=0, z=defaultZ로 설정하고 edge.axis를 rotation으로 사용합니다.

6. 속도 계산 상세 (speedCalculator)

File: src/common/vehicle/physics/speedCalculator.ts

6.1 calculateNextSpeed

// deceleration이 -Infinity이면 즉시 정지
if (deceleration === -Infinity) return 0;

// 가속 또는 감속 중 하나만 적용 (동시 X)
const appliedAccel = deceleration < 0 ? deceleration : acceleration;
let nextVelocity = currentVelocity + appliedAccel * delta;

// maxSpeed clamp (LINEAR vs CURVE)
const maxSpeed = isCurve ? config.curveMaxSpeed : config.linearMaxSpeed;
if (nextVelocity > maxSpeed) nextVelocity = maxSpeed;
if (nextVelocity < 0) nextVelocity = 0;

6.2 Max Speed 분리

Edge 타입Max Speed 소스
LINEARconfig.linearMaxSpeed
CURVE (모든 곡선)config.curveMaxSpeed

6.3 감속 우선순위

1. Sensor (hitZone) → 충돌 감지에 의한 감속 2. Curve Pre-Brake → 전방 곡선 진입 대비 3. Merge Pre-Brake → 합류점 LOCK_WAIT 대비 → Math.max(|sensorDecel|, |curveDecel|, |mergeDecel|) → 가장 강한 감속이 최종 적용

6.4 Brake Distance 공식

calculateBrakeDistance

d = (v0² - v1²) / (2 * |a|)

v0: 현재 속도 (m/s)
v1: 목표 속도 (m/s)
a:  감속도 (음수, m/s²)

calculateMaxSpeedForDistance

v = sqrt(vTarget² + 2 * |a| * d)

주어진 거리에서 도달할 수 있는 최대 속도
감속이 가능한 범위 내에서의 상한선

6.5 Delta Time Clamping

프레임 간 시간 차이(delta)는 MovementUpdateContext.clampedDelta로 전달됩니다. 시뮬레이션 엔진에서 최대 0.1초로 clamping하여 큰 프레임 드랍 시 물리 계산이 발산하는 것을 방지합니다.

7. Edge Ratio 의미

Edge (distance = D) ratio=0 ════════════════════════════════ ratio=1 from_node to_node ├── ratio 0.5 → 중간점 └── ratio > 1 → overflow (다음 Edge로 전환 필요)

ratio 계산

rawNewRatio = currentRatio + (velocity * delta) / edge.distance

ratio → World Position 변환 흐름

rawNewRatio │ ├─ < 1.0 → 동일 Edge 내 이동 │ targetRatio 체크 → 도달 시 ratio=targetRatio, v=0 │ └─ >= 1.0 → Edge Transition overflowDist = (ratio - 1) * edge.distance nextRatio = overflowDist / nextEdge.distance (Multi-hop: while nextRatio >= 1) │ finalRatio (0~1 범위, 최종 Edge 기준) │ └─ interpolatePositionTo(edge, ratio) LINEAR: 직선 보간 → x, y CURVE: renderingPoints 세그먼트 보간 → x, y, rotation │ commitVehicleState → SharedArrayBuffer 기록

1-based Edge Index

CURRENT_EDGE 및 NEXT_EDGE_0~4는 1-based 인덱스입니다. 0은 invalid sentinel 값입니다. 배열 접근 시 edgeArray[edgeIndex - 1]을 사용합니다.

8. Zero-GC 최적화

Movement 시스템은 모듈 레벨 Scratchpad 객체를 재사용하여 프레임마다 새 객체를 생성하지 않습니다. 10만대 이상의 차량을 60fps로 처리할 때 GC pause를 방지하는 핵심 패턴입니다.

Scratchpad 목록

Scratchpad파일용도
SCRATCH_PHYSICSvehiclePhysics.tsPhase 1 결과 (velocity, ratio, edge, shouldSkip)
SCRATCH_VEHICLE_TRANSITIONvehicleTransition.tsPhase 2 결과 (finalEdgeIndex, finalRatio, activeEdge, finalVelocity)
SCRATCH_TRANSITIONvehicleTransition.ts내부 Edge 전환 결과 (handleEdgeTransition 출력)
SCRATCH_TARGET_CHECKvehicleTransition.ts동일 Edge target 도달 체크
SCRATCH_VEHICLE_POSITIONvehiclePosition.tsPhase 3 결과 (x, y, z, rotation, ratio, velocity)
SCRATCH_POSvehiclePosition.ts내부 보간 결과 (interpolatePositionTo 출력)

패턴

// 모듈 최상위에 한 번만 생성
export const SCRATCH_PHYSICS: VehiclePhysicsResult = {
  newVelocity: 0,
  rawNewRatio: 0,
  targetRatio: 0,
  currentEdgeIndex: 0,
  currentEdge: null as unknown as Edge,
  shouldSkip: false,
};

// 매 차량 루프에서 재사용 (new 없음)
calculateVehiclePhysics(ctx, i, data, ptr, SCRATCH_PHYSICS);
// SCRATCH_PHYSICS.newVelocity, .rawNewRatio 등 직접 읽기

주의사항

  • Scratchpad는 단일 스레드(Worker) 내에서만 사용 — 동시성 문제 없음
  • 다음 차량 루프 시작 시 이전 값이 자동으로 덮어쓰기됨
  • activeEdge 같은 참조 타입 필드는 Edge 배열의 요소를 가리킴 (복사 아님)
  • 함수가 out 파라미터로 scratchpad를 받아 결과를 직접 기록 — return 값은 같은 참조

메모리 흐름 요약

updateMovement() for (each vehicle) { SCRATCH_PHYSICS ← Phase 1 결과 기록 SCRATCH_VEHICLE_TRANSITION ← Phase 2 결과 기록 (SCRATCH_PHYSICS 참조) SCRATCH_VEHICLE_POSITION ← Phase 3 결과 기록 (TRANSITION 참조) commitVehicleState ← Float32Array에 최종 기록 } // 루프 끝: scratchpad 내용은 마지막 차량 값 (사용 안 함)
