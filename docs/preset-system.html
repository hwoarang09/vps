Parameter Preset SystemJSON 프리셋으로 실험 환경 일괄 설정

Overview

public/config/parameterMap/에 JSON 파일로 프리셋을 정의하면, UI에서 클릭 한 번으로 맵 + Fab Grid + 차량 + 라우팅/이동/Lock/Transfer 파라미터를 일괄 적용합니다.

Application Flow

PresetPanel에서 클릭 ↓ applyPreset(preset) ├─ 1) fabConfigStore에 파라미터 세팅 │ (movement, lock, routing, transfer, vehInit) ├─ 2) Per-fab routing overrides 적용 ├─ 3) Fab 그룹 세팅 (useFabStatsUIStore) ├─ 4) persistedConfig 업데이트 (fabCountX/Y, numVehicles) └─ 5) requestSettingChange(settingId) ↓ 기존 VehicleTest 플로우 자동 실행 ├─ disposeShmSimulator() — 기존 시뮬 정리 ├─ loadCFGFiles(mapName) — 맵 로드 ├─ applyFabGrid(X, Y) — Fab Grid 생성 └─ setIsTestCreated(true) — 차량 생성 + 렌더링

Preset JSON Schema

모든 필드는 optional. 없는 필드는 기본값 사용.

{
  "name": "BPR+EWMA 비교 실험",
  "description": "설명...",

  // 레이아웃 (맵/fab/차량)
  "settingId": "Y_SHORT",       // testSettingConfig의 ID (맵 결정)
  "numVehicles": 22000,         // 총 차량 수
  "fabCountX": 5,               // Fab Grid 열
  "fabCountY": 22,              // Fab Grid 행

  // 시뮬레이션 파라미터 (Global)
  "movement": {
    "linear": { "maxSpeed": 5.0, "acceleration": 2.0, ... },
    "curve": { "maxSpeed": 1.0, "acceleration": 1.0 }
  },
  "lock": {
    "waitDistanceFromMergingStr": 2.9,
    "requestDistanceFromMergingStr": 5.1,
    "grantStrategy": "BATCH"
  },
  "routing": {
    "strategy": "BPR",
    "bprAlpha": 4, "bprBeta": 8, "bprGamma": 0.2
  },
  "transfer": {
    "enabled": true,
    "mode": { "idlePolicy": "RANDOM_WALK" },
    "rate": { "mode": "utilization", "utilizationPercent": 50 }
  },
  "vehInit": {
    "mode": "equal", "seedMode": "random", "seed": 12345
  },

  // Per-Fab Routing Overrides
  "fabRoutingOverrides": {
    "0": { "strategy": "DISTANCE" },
    "1": { "strategy": "EWMA", "ewmaAlpha": 0.05 },
    "10": { "strategy": "BPR", "bprAlpha": 0.5, "bprBeta": 2, "bprGamma": 0 }
  },

  // 통계 그룹 (FabStats에서 사용)
  "groups": [
    { "name": "DISTANCE", "fabs": [0] },
    { "name": "EWMA", "fabs": [1,2,3,4,5,6,7,8,9] },
    { "name": "BPR c=0", "fabs": [10,11,...,34] }
  ]
}

앱 시작 시 initPresetFromDb()가 마지막 선택 프리셋을 자동 복원.

파라미터 메뉴에서 수동 변경하면 IndexedDB는 안 건드림 → 프리셋 선택 상태 유지.

File Structure

public/config/parameterMap/
├── index.json          ← 프리셋 파일 목록 ["default.json", "bpr-test.json"]
├── default.json        ← 기본값 프리셋
└── bpr-test.json       ← BPR+EWMA 비교 실험 (5×22=110 fabs)

Adding a New Preset

  1. public/config/parameterMap/에 JSON 파일 생성
  2. index.json 배열에 파일명 추가
  3. 끝. UI에서 Operation → Preset 클릭하면 바로 보임.

Preset → Worker Sync

프리셋은 Zustand store만 업데이트합니다. Worker에 적용되는 시점:

  • settingId 있을 때: requestSettingChange() → VehicleTest가 시뮬 재생성 → Worker init 시 store 값 반영
  • settingId 없을 때: store 값만 변경. 이미 돌고 있는 시뮬에는 UI Params 패널에서 "실시간 반영" 해야 함.
