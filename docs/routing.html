Routing SystemDijkstra, BPR, EWMA — 동적 경로 탐색

Three Routing Strategies

StrategyCost FunctionCharacteristic
DISTANCE t0 = distance / maxSpeed 정적. 혼잡 무시. 캐시 가능.
BPR t0 * (1 + α * (V/C + γ)^β) 혼잡도 기반 동적 비용. 매 탐색마다 재계산.
EWMA EWMA(α * new + (1-α) * old) 실측 통과시간 기반. 과거 데이터 가중 평균.

BPR (Bureau of Public Roads) Model

Formula

cost = t0 * (1 + α * (V/C + γ)^β)

t0 = freeFlowTime = edge.distance / maxSpeed
V  = edgeVehicleQueue.getCount(edge)    // 현재 차량 수
C  = floor(edge.distance / vehicleSpacing) // 용량
γ  = ratio offset (기본 0.2)

Parameters

ParamNameDefaultEffect
α (alpha)혼잡 가중치4클수록 혼잡 시 비용 증가 폭 큼
β (beta)비선형 지수8클수록 용량 근접 시 급격히 비용 증가
γ (gamma)ratio 오프셋0.2V/C ≤ 1일 때 β 효과 보정

Why gamma(γ)?

문제: V/C ratio가 항상 ≤ 1이면, ratio^β는 β가 클수록 오히려 작아짐.

해결: (V/C + γ)로 오프셋을 주면 ratio가 1을 초과할 수 있어, β가 클수록 혼잡 비용이 정상적으로 증가.

예: V/C=0.8, γ=0.2 → (0.8+0.2)^β = 1^β = 1 (기준점)

EWMA (Exponential Weighted Moving Average)

ewma_new = α * observed_transit_time + (1 - α) * ewma_old

α = 0.1 (기본) → 과거 90%, 최신 10%
α = 0.5 → 과거 50%, 최신 50% (민감)
α = 0.01 → 매우 안정적 (변화 느림)

Cold start: 데이터 없으면 freeFlowTime으로 시드.

Dijkstra Algorithm

Implementation

  • Graph: Edge 기반 (1-based index). 각 Edge의 nextEdgeIndices가 연결 관계.
  • Heap: 커스텀 MinHeap — 단일 number[], 인덱스 연산, O(E log V)
  • Cache: LRU 2000개 (DISTANCE만 캐시. BPR/EWMA는 동적이라 캐시 안 함)
  • Scratchpad: distArray, prevArray 사전 할당 재사용 (GC 없음)

Per-Fab Routing Config

각 Fab은 독립된 라우팅 설정을 가질 수 있습니다. Global 설정을 기반으로 Fab별 오버라이드.

// fabConfigStore
routingConfig (global): { strategy: "BPR", bprAlpha: 4, bprBeta: 8, bprGamma: 0.2 }

fabOverrides[0]: { routing: { strategy: "DISTANCE" } }           // Fab 0: DISTANCE
fabOverrides[1]: { routing: { bprAlpha: 2, bprBeta: 4 } }       // Fab 1: BPR a=2 b=4
fabOverrides[9]: { routing: { strategy: "EWMA", ewmaAlpha: 0.3 } } // Fab 9: EWMA
